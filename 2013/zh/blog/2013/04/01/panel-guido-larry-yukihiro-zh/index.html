Guido、Larry、Yukihiro 聯手參加 PyConTW 座談

2013.04.01 by marr

想要見三巨頭嗎? 趕快到這裡報名!

對於 PyCon Taiwan 的邀請，Larry Wall 率先回覆答應，他提到，Perl 6 除了唐鳳實作的 Haskell 版本外，也考慮有個 Forth 版本，此次能到亞洲地區，期待這股東方禪意能為 Perl 的新方向帶來啟發。

Python 跟 Perl 之前已有良好的合作經驗，Ruby 在 web 和 MATSUE project 的成功，則引起 PyCon Taiwan 籌辦人員的興趣，認為三大巨頭的同場交流，肯定會是 Scripting 語言歷史的重要一頁。Guido van Rossum 在聽聞 Larry Wall 和 Yukihiro Matsumoto 同意參加 PyCon Taiwan 活動後，認為這是同台切磋的難得機會，終於排除萬難，促成這次的高峰會談。

對於 Python 在 web 的發展，Guido 相當關心，他說，front-end 部份正關注 brython 的進展，對於 framework 比 keyword 更多的現象，他鼓勵理念相近的 framework 進行整併，例如 Pylons 後繼成為 Pyramid，命名也符合 Pythonic 原則，值得借鏡。不過，Guido 還是不願鬆口，欽點某個 web framework 來一統天下，他說，這是個性使然，此次和 Yukihiro Matsumoto 一起相聚於台灣，就是想當面請教 Ruby 獨尊 Rails 帶來的利弊。

最後，告訴大家兩個秘辛。Guido 雖然熱衷參加 PyCon 活動，但犯太歲不宜遠行，使得他對今年的台灣行有點猶豫。話說，沒有太多人留意到 Guido 本身是棒球愛好者，基於這個事實，Guido 當然沒錯過經典賽的點滴。荷蘭隊被日本隊狂電的那場比賽，Guido 早有預感，終究 今年是太歲年，只是 Guido 沒想到威力這麼大。東京球場是 Ruby 愛好者的主場，再加上全球 Scala 愛好者的球迷，都在集氣為日本隊加油，打得辛苦自是難免。

同上，更不為人知的秘密是，Guido 本身是荷蘭代表隊的隨隊人員之一，他低調隨著荷蘭隊來過台灣。有圖有真相。

僅管 Perl、Python、Ruby 各有專擅，不過，沒有人知道使用它們來唱饒舌歌 (Java 有個 We Code Hard 版本)，會是怎樣味道。別忘了到 PyCon Taiwan 2013 的座談會裡，聽聽 Larry、Guido、Yukihiro 唱給你聽!
